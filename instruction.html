Instruction + FAQ

Manuals

Please check the iPWS Cup 2023 competition manual for details on how to use this competition site.

There are some useful scripts, such as sample anonymization and attack scripts and data format validator, available in iPWS Cup 2023 GitHub repository for your reference.

You might also find some useful information on the CodaLab official document.

FAQ

[Q1] I tried to submit my data, but I got an error and was unable to upload my data.
[A1] First, please check the manual to see if you are following the instruction correctly.

  • Only the zipped file is accepted for submission.
  • Files in the zip file should be laid flat; no folder is allowed.
  • If you are using MacOS, make sure that __MACOS folder and an extra directory are not included in the zip file. We highly recommend you use the zip command in the terminal for creating the zip file. Using the GUI feature is not recommended.
  • If you can't resolve the issue by yourself, contact the iPWS Cup committee secretary:
    secretary-ipwscup2023(at)iwsec.org (replace (at) with @).

[Q2] Why you are distributing multiple original health data when I can only submit one anonymized health data?
[A2] To prevent unfairness in the competition.
We are using the synthetic data generated from the NHANES 2017-2018 dataset for the original health data. Although we will try our best not to put any bias in the synthetic data, there is a very good chance that some synthetic data will be easier to anonymize than others. We decided to distribute multiple sets of original health data and let you decide the one to anonymize so as to mitigate the unfairness.

[Q3] Should the row index should start from 0 or 1 when I create the estimated index file at the attack phase?
[A3] The count of the row index should start from 0, not 1. For example, you should set 0 if you want to specify the first row of the original health data.

[Q4] Can my teammate apply his/her account to the competition?
[A4] We allow only one account per team in the competition. This rule is set because we need to align the maximum number of allowed submissions among teams; the CodaLab only allows us to set the max limit per user account, not per team.

[Q5] We are struggling with the data submission. Are there any useful tools?
[A5] Yes, you can find some tools in the iPWS Cup 2023 GitHub repository, such as:

  • checkC.py: Data format validator for anonymized health data.
  • checkE.py: Data format validator for an estimated index file.
  • rlink2.py: Sample re-identification script that will generate an estimated index file in the correct data format.

[Q6] The leaderboard shows that some teams are making more than 10 entries in the attack phase. Why is this?
[A6] The number of entries shown in the leaderboards count both succeeded and failed submissions, so the number goes over 10 for some teams. The number of successful submissions is correctly limited to 10.
